# SRGAN Model

A PyTorch implementation of **SRGAN** for image super-resolution.  
This project trains a Super-Resolution Generative Adversarial Network to convert low-resolution images into higher-resolution outputs using a generator, discriminator, perceptual loss, adversarial loss, and pixel-level reconstruction loss.

The repository includes training, testing, pretrained model weights, loss graphs, and evaluation utilities for super-resolution experiments.

---

## Overview

Single-image super-resolution is the task of reconstructing a high-resolution image from a low-resolution input image. Traditional upscaling methods often produce blurry results because they focus mainly on pixel-wise similarity. SRGAN improves this by using a GAN-based architecture that encourages visually realistic high-frequency details.

This project uses:

- **Generator** to produce super-resolved images from low-resolution inputs
- **Discriminator** to distinguish generated images from real high-resolution images
- **VGG19-based perceptual loss** to preserve visual features
- **MSE/L2 loss** for pixel-level reconstruction
- **Adversarial loss** for photo-realistic texture generation
- **Total variation loss** as an optional smoothing regularizer

## Model Architecture

### Generator

The generator takes a low-resolution RGB image and outputs a super-resolved image. It contains:

1. Initial convolution layer
2. Multiple residual blocks
3. Skip connection after residual learning
4. PixelShuffle-based upsampling blocks
5. Final convolution with `Tanh` activation

The default generator configuration uses:

- 3 input image channels
- 64 feature maps
- 16 residual blocks
- Scale factor of 4

---

### Discriminator

The discriminator classifies whether an image is real high-resolution or generated by the SRGAN generator. It uses:

1. Convolution layers
2. LeakyReLU activations
3. Batch normalization
4. Downsampling discriminator blocks
5. Fully connected layers
6. Sigmoid output for real/fake prediction

---

## Training Pipeline

Training is divided into two stages.

### 1. Pre-training Stage

The generator is first trained using pixel-wise **L2/MSE loss**.  
This helps the generator learn basic low-resolution to high-resolution reconstruction before adversarial training begins.

Default:

```text
pre_train_epoch = 100
```

---

### 2. Fine-tuning Stage

After pre-training, the model is trained using a combination of:

- VGG19 perceptual loss
- Adversarial loss
- L2 reconstruction loss
- Optional total variation loss

Default:

```text
fine_train_epoch = 50
```

During fine-tuning:

- The discriminator learns to separate real high-resolution images from generated images.
- The generator learns to produce outputs that look realistic and preserve perceptual image features.

## Dataset Format

The model expects paired low-resolution and high-resolution image folders.

Example:

```text
custom_dataset/
│
├── low_res/
│   ├── image_001.png
│   ├── image_002.png
│   └── ...
│
└── high_res/
    ├── image_001.png
    ├── image_002.png
    └── ...
```

Important:

- File ordering is handled using sorted filenames.
- Low-resolution and high-resolution images should correspond by filename/order.
- Images are loaded as RGB.
- Pixel values are normalized to the range `[-1, 1]`.

## Usage

The main script is:

```bash
python main.py
```

It supports three modes:

```text
train       Train SRGAN
test        Test SRGAN with ground-truth images and calculate PSNR
test_only   Run inference on low-resolution images only
```

---

## Training

Run training with paired low-resolution and high-resolution folders:

```bash
python main.py \
  --mode train \
  --LR_path ./custom_dataset/low_res \
  --GT_path ./custom_dataset/high_res \
  --batch_size 16 \
  --res_num 16 \
  --scale 4 \
  --patch_size 24 \
  --pre_train_epoch 100 \
  --fine_train_epoch 50
```

### Training Arguments

| Argument | Description | Default |
|---|---|---|
| `--LR_path` | Path to low-resolution image folder | Local Windows path in code |
| `--GT_path` | Path to high-resolution image folder | Local Windows path in code |
| `--res_num` | Number of residual blocks in generator | `16` |
| `--num_workers` | Number of DataLoader workers | `0` |
| `--batch_size` | Training batch size | `16` |
| `--L2_coeff` | L2 loss coefficient | `1.0` |
| `--adv_coeff` | Adversarial loss coefficient | `1e-3` |
| `--tv_loss_coeff` | Total variation loss coefficient | `0.0` |
| `--pre_train_epoch` | Generator pre-training epochs | `100` |
| `--fine_train_epoch` | GAN fine-tuning epochs | `50` |
| `--scale` | Super-resolution scale factor | `4` |
| `--patch_size` | Low-resolution crop patch size | `24` |
| `--feat_layer` | VGG19 feature layer for perceptual loss | `relu5_4` |
| `--vgg_rescale_coeff` | VGG perceptual loss scaling coefficient | `0.006` |
| `--fine_tuning` | Whether to load a pretrained generator | `False` |
| `--in_memory` | Load full dataset into memory | `True` |
| `--generator_path` | Path to pretrained generator checkpoint | None |
| `--mode` | Execution mode: `train`, `test`, or `test_only` | `train` |

## Fine-tuning From a Pretrained Generator

To continue training from an existing generator checkpoint:

```bash
python main.py \
  --mode train \
  --LR_path ./custom_dataset/low_res \
  --GT_path ./custom_dataset/high_res \
  --fine_tuning true \
  --generator_path ./p_model_350.pt
```

## Testing With Ground Truth

Use `test` mode when both low-resolution and ground-truth high-resolution images are available.

```bash
python main.py \
  --mode test \
  --LR_path ./custom_dataset/low_res \
  --GT_path ./custom_dataset/high_res \
  --generator_path ./p_model_350.pt
```

This will:

- Generate super-resolved images
- Save results in the `result/` folder
- Compute PSNR values
- Save PSNR output in `result.txt`

## Evaluation

### PSNR

The project includes a PSNR script:

```bash
python psnr.py
```

PSNR measures pixel-level similarity between the original high-resolution image and the generated super-resolved image. Higher PSNR usually indicates better reconstruction quality.

---

### LPIPS

The `ssim.py` file currently computes LPIPS distance using an AlexNet-based LPIPS model.

```bash
python ssim.py
```

LPIPS is a perceptual similarity metric. Lower LPIPS values indicate that two images are perceptually more similar.

Note: despite the filename `ssim.py`, the script currently calculates LPIPS, not SSIM.

## Common Issues

### 1. FileNotFoundError for dataset path

Make sure the paths passed to `--LR_path` and `--GT_path` exist.

Correct example:

```bash
python main.py --mode train --LR_path ./data/low_res --GT_path ./data/high_res
```

---

### 2. Missing `model/` or `result/` folder

Create them manually:

```bash
mkdir model
mkdir result
```

---

### 3. CUDA not available

The code automatically uses CUDA if available, otherwise CPU:

```python
torch.device("cuda" if torch.cuda.is_available() else "cpu")
```

Training on CPU will be much slower.

---

### 4. `compare_psnr` error from scikit-image

Newer versions of `scikit-image` removed `compare_psnr`.

Use the older version listed in `environment.yml`, or update the import in `mode.py`:

```python
from skimage.metrics import peak_signal_noise_ratio
```

Then replace:

```python
compare_psnr(...)
```

with:

```python
peak_signal_noise_ratio(...)
```
